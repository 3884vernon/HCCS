Barcode & Duplicate Analysis

Anchors barcodes to the NNK·NNK region instead of absolute read positions — robust to Nanopore length variation. Set the upstream flanking sequence in the config above to find NNK reliably. Plate barcode starts a fixed distance upstream of NNK; UMI starts a fixed distance downstream.

UDI → Quadrant Mapping (for Sort-Seq ΔF/F enrichment)

Click Scan Files for UDIs after loading FASTQs above. The analyzer samples the first 5,000 reads of each file to fingerprint the dominant UDI. Then label each file's UDI with a quadrant name (Q1–Q4) and ΔF/F gate to enable cross-quadrant enrichment scoring.

How it works: For each read, the analyzer locates NNK·NNK using the flanking sequence (upper config) and validates that the 6 bp downstream of the flank match the NNK pattern NN[GT]NN[GT]. If validation fails or the flank isn't found, RC mode tries the reverse complement — this correctly handles paired-end Illumina R2 reads and palindromic flanks like TGAGCTCA. Once anchored, plate barcode is read from NNK_start - plate_starts for plate_length bp; UMI is read from NNK_end + umi_starts for umi_length bp. Fuzzy UMI match groups duplicates whose UMIs differ by ≤N bases.
